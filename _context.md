## Key Business Rules

### Despatch Orders
- **Partial lock on delivered orders** — docs/freight sections remain editable after delivery
- **Draft status** suppressed on print/PDF output
- **Rate card locking** — orders linked to `rate_card_id` (not vendor/transporter pair) to correctly scope edit/delete locks

### Agent Commissions
- Slab-based auto-calculation triggered on `Delivered` status
- Supports partial payments

### Transporter Rate Cards
- History maintained — old rate marked `Inactive` when new rate is added

### Transporter Pending Bills
- Three sections in pending bills view

## Known Patterns & Conventions
- **File editing:** Copy from server → edit locally → upload back via cPanel File Manager
- **Verify before editing:** Always use `sed -n '[range]p'` to confirm file content before str_replace
- **One fix at a time** — avoid changes that ripple into other modules
- **Complete file preferred** — JPS prefers receiving the full updated file for direct upload
- **String replacement failures** have been a recurring issue — always verify with `sed -n` before and after

---

## Critical Lessons Learned
1. **Nested form bug** — Data Cleanup form was accidentally nested inside Company Settings form causing accidental data deletion. Always isolate destructive actions in separate forms with explicit confirmation (type "DELETE").
2. **Challan sequence skipping** — Two separate generation calls caused gaps. Fixed with single atomic `generateChallanAndDespatchNo()`.
3. **Rate card false positives** — Linking to `rate_card_id` (not vendor/transporter pair) fixed incorrect lock scope.
4. **cPanel SMTP** — Port 587 intercepted by host; SSL peer verification must be disabled via `SMTPOptions`.
5. **Wrong file version** — Many failed fixes caused by editing stale/wrong folder files. Always confirm exact content first.
